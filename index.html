Dap.js: Declarative script for massively interactive web-applications.

Why dap?

Natively reactive

Reactivity is the base concept of dap. No need to manually track changes in your application's status, or even bother to bind update events — all dependencies are resolved naturally under the hood. Dap renders and updates all the dependent elements interactively, based on their dap-rules. 

'sample'.d("$choice="	// make a {DIV class="sample"} element, and in its scope define a $choice status variable 
	,'B.selected'.d("! title ($choice not-yet)?")	// print a message with $choice, mixed with some text from dictionary
	,'UL.options'.d("* fruit"	// make UL element, and for each row in fruit dataset, repeat:
		'LI'	.d("! .fruit")	//  make LI element, show fruit value for given row, and
			.ui("$choice=.fruit") // on UI event (click by default) update the $choice status variable
	)
)
.DEF({
	fruit:["apple","orange","tomato","banana"], // rowset data is a usual array
	title:"My favorite fruit is ",	// messages are defined here, in dictionary - not inside the rules
	"not-yet":"what?"	// hyphens and other symbols are okay for entry names, as long as they are quoted
})

Clear, concise and observable

Dap syntax is designed to focus on data and logic, instead of syntactic decoration. Dap application may be times smaller than similar code in other reactive environments. You can well observe your code without digging through sparse lines of curly brackets and boilerplate code.

'H1'.d("! `Hello,world!")

Simple to start, easy to advance

A whole lot of magic can be done with the few "stantard" dap operations. But when it's time to go beyond — you can easily import 3-rd party extensions or expand dap with your own custom operations.

Fast and lightweight

Dap will not introduce any noticeable latency to your Web UI. The dap support script weighs less than 20kiB, dap-rules are parsed lazily, DOM manipulations are precise and deterministic, and only involve affected elements.

Flexible and expandable

Code examples

Below are a few simple examples of how easily data/event-driven applications can be built with dap. Examples below are inspired by the examples from other reactive frameworks, such as React.js, Angular, etc. so if you're familiar with any of those, you can compare the ideas and approaches behind dap and other frameworks.

Important notice: unlike some other frameworks, dap does not imply any specific directory structure organization for your application. Simple web-applications can easily consist of a single, static HTML file, maybe accompanied with a CSS stylesheet or other usual stuff, plus any related data sources. For more complex projects, organization is up to the developer.
